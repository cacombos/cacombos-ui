How to contribute?

Please help us build better website by sending your device UE Capability or any log file includes Carrier Aggregation and Dual Connectivity combinations. We will publish supported combinations on our website.

1. Get Carrier Aggregation combinations from your device

There is many way to get Carrier Aggregation combinations from devices. Here are instructions for mobile phones.

This method should work every Qualcomm Android-devices and it require root.

Download and extract platform-tools to your computer. On Linux you can install this also via package manager.

Open your computer Terminal. On Windows use PowerShell or CMD and run command to on platform-tools directory.

Connect your device with USB to your computer. You have to enable developer options in Android settings and USB-debugging.

now with adb shell, download Diag.cfg configuration file to diag_logs directory.

For new SDX53/60 devices you may also have to create another directory before logging with command mkdir -p /data/vendor/ramdump/diag_logs/

Now we are ready to start logging. You have to now use su to get root-permission and then start logging with diag_mdlog command

During logging, put your phone to Airplane Mode on and then off and wait two minute so your phone will connect back to network. Use keyboard combo CTRL + C to end logging on terminal/power shell/cmd. The log file will be saved to /sdcard/diag_log folder.

Scat will save the modem log in qmdl format (Qualcomm only) and in pcap format (Qualcomm and Exynos).

Install on Debian/Ubuntu

Install prerequisite packages from python pip package manager, then clone scat from Github

Install on macOS

Run these commands in MacOS terminal.

Extra steps for M1
  • On M1, we need a compatibility library to support usb enumeration, but this can only be installed with Homebrew in Rosetta
  • Open Finder and navigate to Applications -> Utilities
  • Right-click on the "Terminal" app and choose Duplicate
  • Rename the duplicated "Terminal" app to "Terminal x86_64"
  • Right-click the Terminal x86_64 app, then click on "Get Info"
  • Select the option to "Open using Rosetta"
  • Launch from here or drag to the dock for a shortcut
  • Install Homebrew from within the x86_64 emulated terminal
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  • Install libusb and lsusb with brew to communicate with your device over USB. You have to install libusb with libusb-compat to make sure this work with your M1.
    brew install libusb libusb-compat lsusb
  • Run lsusb to get your device hub ID
    lsusb

Install on Windows

Install Python from Windows Store and Git

Open PowerShell with Administrator and install prerequisite python packages, then clone the repository.

Enable diag

Connect your phone to your computer with USB and enable with these methods:

Non-root methods

Some devices allow enable USB-diag without root:

  • Samsung devices can enable diag by dialing *#0808# and then enabling DM + RMNET + MODEM (Qualcomm) or RNDIS + ACM + DM + ADB (Exynos). Certain version of firmwares after 2018 blocking the access to hidden menu with code. No solution then without root.
  • OnePlus, Realme and Oppo devices: Enable diag by dialing *#8011#. To disable, disable/enable USB Debugging in Developers Menu.
  • Alcatel/TLC devices can enable diag by dialing ###2324# until the device prompts with Diag Protector OFF.
  • Blackberry devices can enable diag by dialing *#46368676#*#* to enter the Testing-menu and enable RMNET.
Root required methods

These methods should work almost every rooted Android Qualcomm-devices

  • Install adb tools
  • Open terminal or Powershell and run command adb shell
  • Run su to run super user priviledge command
  • Then run command in adb shell:
    • If you have SDX24 or older: setprop sys.usb.config diag
    • If you have SDX50 or SDX55: setprop sys.usb.config diag,diag_mdm,qdss,qdss_mdm,serial_cdev,dpl,rmnet,adb
    • If you have SDX60: setprop sys.usb.ffs.ready 1; setprop sys.usb.configfs 1; setprop sys.usb.config diag,adb

Windows libusbK driver installation

To Windows you have to install libusbK driver. Before install this driver you may have to uninstall Qualcomm diag driver first. Before you can install driver, you have to enable diag.

  • Select Create and install libusb-win32 device driver a package
  • On USB InfWizard, select libusbK
  • Select your phone interface. Some phones show multiple interface, It's same interface as you use for QPST example.
  • Select Client Installer and path where driver will be saved.
  • Now driver is created and you can start install by clicking Finnish / Install Driver Now.
  • Click next to install driver to your computer. You can see installed driver in Device Manager

Run Scat

Then use scat to record modem data to qmdl and pcap file. Use lsusb command on Linux/MacOS to set device address parameter, example -a 001:010. The first part before colon (:) is Bus, what is normally 001, or 002 and then second part after colon is USB device id. Check other parameters in scat README.md file.

Before 2020 released devices

Doesn’t work with Android One devices. This works only with Qualcomm Devices before SD865.

  • Open Android Settings -> About phone and click Kernel version 5 times. This will open CIT menu
  • On CIT menu open Additional tools on dropdown menu on right top
  • Go to Modem test tools -> Open/Close Offline Log to open logs
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected to 4G LTE network.

  • Enter to the file manager and find the /sdcard/dialog_logs/XXXX/mdm/diag_log file (XXXX is some generated timestamp value). The log file file extension should be qmdl or qmdl2.

  • Go to Android Settings -> Service & feedback
  • Open additional menu on right top -> Generate logs
  • Go to "Find Phone, signal and mobile networks" section and press Generate log.
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected to 4G LTE network.
  • Go back to Logs menu and tap “Finish and upload”. You don't need continue send feedback to Xiaomi, you can close now logs menu.
  • Enter to the file manager and find the /sdcard/dialog_logs/XXXX/mdm/diag_log or /sdcard/dialog_logs/XXXX file (XXXX is some generated timestamp value). The file extension should be on .bin. If not exist then extension should be qmdl or qmdl2.

Qualcomm - Old method

  • Call to *#546368#*#modelnumber#. (Replace model number of your phone model number. You can find model number on Android Settings -> About.
  • Open Field Test -> LGODM Logging(LGODL).
  • Click Start to start logging
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected to 4G LTE network.
  • Click Stop to end logging
  • Now you should find log File Manager -> lgodm -> logs.

Qualcomm - New method

  • Dial to 277634#*#
  • Open Log Service -> Modem log to enable modem logging.
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected 4G LTE network.
  • Disable Modem logging via same menu.
  • Now you should find log File Manager -> lgodm -> logs.
  • Call to *#*#3248665#*#*.
  • Go to 4. Modem Conf.(DbgCfgTool Orginal)
  • Enable diag logging
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected to 4G LTE network.
  • Stop logging
  • Copy log to your storage with Copy diag log.
  • Now you can find log file on File Manager -> diag_log_xxxx -> log -> diag_log_XXXXXX.qmdl

Exynos

This method require ShannonDM software on Windows. Please find this program from internet or use Scat.

  • Connect phone to your computer with USB
  • Dial to *#0808# and enable RNDIS + ACM + DM + ADB and click OK to save settings. Don't reboot your phone.
  • Open ShannonDM on Windows.
  • Open Preference (blue) and click Auto Detect Modem Interface (red) and Apply settings and OK.
  • Now you can start record with recording button (red). Shannon will ask where to save log file.
  • Switch your device to Airplane Mode, wait few second and turn Airplane mode off. Wait some time to get connected to 4G LTE network.
  • Stop logging with same button you started recording. Wait few minutes so Shannon complete writing log file.

Qualcomm

This method will extract 00028874 and nvitem from 4G-devices and UE Capability Information 4G and 5G devices. Please prefer Scat-logging.

  • Dial to *#9900# and then SysDump menu should open.
  • Press Run CP based log and then Copy to sdcard (include CP Ramdump).
  • You should find BandInfoXXX file on Internal Storage: /sdcard/logs/err/bandinfo_XXXX . Also you should find UECapabilityInformation DLF file in same directory.
In 5G-devices the BandInfo file include only Carrier Policy defined combinations. UECapabilityInformation DLF file will include combinations requested by network.

This logging only works iOS 11+

Start logging

  • Download baseband profile and install on the iOS device. This profile will expire after 21 days.
  • Restart device
  • Put your phone to Airplane mode on and off. Wait some time your phone connect to network.
  • Trigger saving log with pressing and release both Volume buttons + Side (or Top) button. On an iPhone you will feel a short vibration when a sysdiagnose is successfully triggered.
  • Wait 10min for the log gathering complete.
  • Connect your device to you computer and then iTunes should copy log files automatically to your computer.

Log locations

Now you can find the saved log file in AppData on Windows or Libraries on macOS, here instructions to access hidden folders.

macOS

~/Library/Logs/CrashReporter/MobileDevice/[Your_Device_Name]/Baseband/
In this directory you should find qmdl log file. The log files are splitted to multiple 1MB files. If you have iPhone 12/12 Pro/12 Pro max the log file is in new qdss format and log file extension is .bin.

Windows

%AppData%\Apple Computer\Logs\CrashReporter\MobileDevice\[Your_Device_Name]\Baseband\
In this directory you should find qmdl log file. The log files are splitted to multiple 1MB files. If you have iPhone 12/12 Pro/12 Pro max the log file is in new qdss format and log file extension is .bin.

If you have Intel version of device then logging extensions is istp. You can also try submit these files to us and we could try parse and find combinations from Intel modem files.

Stop logging

To stop logging you have to remove baseband profile from your device.

  • Open Settings
  • Go to General -> Profiles
  • Tap Remove and enter passcode if asked
  • Restart the device
